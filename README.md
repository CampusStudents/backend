# Backend CAMPUS

Платформа для поиска тиммейтов среди студентов под хакатоны, учебные проекты, стартапы и внутривузовские мероприятия.

**Основной фокус** — быстрый подбор команды с учётом навыков, ролей, дедлайна, города и формата (онлайн/оффлайн).

## О проекте

Студенты часто не успевают собрать команду под хакатон или курсовой проект из-за дедлайнов и географической разбросанности кампусов.  
Campus Match решает эту проблему через:

- создание проектов с указанием нужных ролей и навыков  
- удобный поиск и фильтрацию  
- базовый scoring-рекомендаций  
- заявки с сопроводительным письмом  
- удобное принятие/отклонение кандидатов + передача ссылки в чат (Telegram/VK/Max и др.)  
- отдельную сущность «Мероприятия» (хакатоны, олимпиады, турниры) с возможностью привязки проекта

### Целевая аудитория
- Студенты 18–25 лет (преимущественно Россия)  
- Участники хакатонов, курсовых/дипломных, стартап-проектов  
- Организаторы мероприятий в вузах  
- Молодые специалисты, ищущие side-проекты для портфолио

## Стек (backend)

- **FastAPI**  
- **PostgreSQL** + **SQLAlchemy** + **Alembic**  
- **Pydantic v2**  
- **Redis** (кэш / celery result backend)  
- **Celery** / **Taskiq** + **Kafka** (асинхронные задачи, уведомления)  
- **Docker** + **Docker Compose**  
- **Nginx** (reverse proxy)  
- **S3**-совместимое хранилище (портфолио, скриншоты)  
- **JWT**-аутентификация

## Быстрый запуск (Docker)

```bash
# 1. Клонировать репозиторий
git clone <repository-url>
cd campus-match-backend

# 2. Создать .env файл (или использовать .env.example)
cp .env.example .env

# Самые важные переменные, которые нужно заполнить/проверить:
# DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/campus
# REDIS_URL=redis://redis:6379/0
# SECRET_KEY=your-very-long-random-secret-key-here
# S3_ENDPOINT_URL=... (если используете внешний S3)

# 3. Запустить
docker compose up -d --build

# 4. Применить миграции (один раз или при изменении моделей)
docker compose exec backend alembic upgrade head

# 5. Открыть документацию API
http://localhost:8000/docs
# или redoc: http://localhost:8000/redoc
```

Остановка:
```bash
docker compose down
# с удалением томов (если нужно полностью очистить):
docker compose down -v
```

## Структура проекта (основные директории)

```
├── alembic/               → миграции базы данных
├── src/
│   ├── api/               → роутеры FastAPI
│   ├── core/              → настройки, security, exceptions
│   ├── db/                → сессии, engine, base модель
│   ├── models/            → SQLAlchemy модели
│   ├── schemas/           → Pydantic схемы (вход/выход)
│   ├── services/          → бизнес-логика
│   ├── repositories/      → доступ к данным
│   └── utils/             → хелперы, scoring, уведомления
├── tasks/                 → celery / taskiq задачи
├── tests/                 → тесты
├── .env.example
├── alembic.ini
├── docker-compose.yml
└── Dockerfile
```

## Полезные команды

```bash
# Логи бэкенда
docker compose logs -f backend

# Зайти в контейнер
docker compose exec backend bash

# Запустить тесты (если настроены)
docker compose exec backend pytest

# Создать новую миграцию после изменения моделей
docker compose exec backend alembic revision --autogenerate -m "added project status"
```

## Текущий статус (MVP)

- Регистрация / авторизация  
- Профиль + навыки + портфолио  
- Создание / редактирование проектов  
- Привязка проекта к мероприятию  
- Создание мероприятий  
- Поиск + фильтры + базовый scoring  
- Заявки в проекты + сопроводительное письмо  
- Принятие / отклонение + поле «ссылка на чат»  
- Уведомления о принятии (email + Telegram-бот)  
- Панель создателя проекта

Готовится / в планах:  
- Swipe-интерфейс (рекомендации)  
- Подписки на организаторов  
- Встроенный чат  
- Авто-дополнение портфолио после завершения проектов
